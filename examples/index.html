Aria Autocomplete
Accessible, extensible, dependency-free autocomplete

Modes

While not strictly necessary, this component is intended to be invoked on an <input /> which will update automatically in the background as autocomplete option(s) are chosen.

Autocomplete Source

A variety of sources can be used for the autocomplete options. These include:

Source as Array of Strings

new AriaAutocomplete(element, {
    source: [
        "Afghanistan",
        "Albania",
        "Algeria",
        ...
    ]
});

Source as Array of Objects

The objects used for the autocomplete source should ideally have value and/or label properties. But if you're unable to transform the source array ahead of use, there is also a sourceMapping option.

new AriaAutocomplete(element, {
    source: [
        {
            code: "AF",
            code3: "AFG",
            name:" Afghanistan"
        }, 
        {
            code: "AL",
            code3: "ALB",
            name: "Albania"
        },
        {
            code: "DZ",
            code3: "DZA",
            name: "Algeria"
        },
        ...
    ],
    sourceMapping: { 'value': 'code3', 'label': 'name' }
});

Element as Source

If the source option is falsy, or is an empty Array, AriaAutocomplete will check to see if the element used is a <select>, or if there are any checkboxes inside the element, and will build up a source Array from the available options. If checkboxes are used, the multiple option will always be set to true.

When an option in the autocomplete is selected, the corresponding option or checkbox will be set to selected/checked as well.

new AriaAutocomplete(element);

Function as Source

If you want to use your own client-side filtering logic, you can also use a function as the source which returns an Array of options to render (that follows the same rules as the Array examples above). In this example, no matter what you search for, 10 random countries will be returned.

new AriaAutocomplete(element, {
    maxResults: 10,
    source: function (value) {
        var arr = countryList.slice();
        for (var i = arr.length - 1; i > 0; i--) {
            var j = Math.floor(Math.random() * (i + 1));
            var temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
        return arr;
    }
});

String (Endpoint) as Source (in progress)

If you want to send the query to an endpoint / your server, you can do that too. The current query, and your chosen value for max results to render, will be added to the url.
E.g. https://some-url-here/endpoint?q=norway&limit=20

new AriaAutocomplete(element, {
    source: 'https://some-url-here/endpoint',
    asyncMaxResultsParam: 'limit',
    asyncQueryParam: 'q',
    maxResults: 10
});

Options

The full list of available options is as follows:

source: Array | String | Function
string for async endpoint, array of strings, array of objects with value and label, or function
sourceMapping: Object
an object detailing the properties to use for the label and value when using an Array of Objects as the source - see Array examples above.
delay: Number
input delay before running a search
Default: 100
minLength: Number
Minimum number of characters to run a search (includes spaces)
Default: 1
maxResults: Number
Maximum number of results to render
Default: 9999
showAllButton: Boolean
Render a button that triggers showing all options
Default: true
confirmOnBlur: Boolean
Confirm current selection (from using arrow keys) when blurring off of the control. Will also check for an exact text match
Default: true
multiple: Boolean
Allow multiple items to be selected
Default: false
maxItems: Number
Maximum number of items that can be selected in multiple mode
Default: 9999
multipleSeparator: String
In multiple mode, if the original element was an input, the character that separates the values
Default: ','
placeholder: String
Placeholder text to show in generated input
Default: 'Type to search...'
noResultsText: String
Text to show (and announce to screen readers) if no results found
Default: 'No results'
listClassName: String
Custom class name to add to the generated list element
Default: ''
inputClassName: String
Custom class name to add to the generated input element
Default: ''
wrapperClassName: String
Custom class name to add to the generated component wrapper
Default: ''
cssNameSpace: String
The string that prepends all internal BEM class names, e.g. 'aria-autocomplete__input'
Default: 'aria-autocomplete'
asyncQueryParam: String
When source is a string, param to use when adding input value to it
Default: 'q'
asyncMaxResultsParam: String
When source is a string, param to use when adding maxResults option + current selected count to it
Default: 'limit'
srDeleteText: String
@description in multi mode, screen reader text used for element deletion - prepended to label
Default: 'delete'
srDeletedText: String
@description in multi mode, screen reader text announced after deletion - appended to label
Default: 'deleted'
srSelectedText: String
@description screen reader text announced after selection - appended to label
Default: 'selected'
srExplanatoryText: String
@description screen reader explainer added to the list element via aria-label attribute
'Search suggestions'
srAssistiveText: String
@description screen reader description used for main input when empty
Default: 'When autocomplete results are available use up and down arrows to review and enter to select. Touch device users, explore by touch or with swipe gestures.'
srResultsText: Function
@description screen reader announcement after results are rendered
Default: length => `${length} ${length === 1 ? 'result' : 'results'} available.`
onAsyncSuccess: Function
Callback after async call completes - is passed the xhr Object, and can be used to format the results by returning an Array
onResponse: Function
Callback prior to rendering - can be used to format the results by returning an Array
onSearch: Function
Callback before search is performed - can be used to affect search value by returning a new one
onSelect: Function
@description callback after selection is made
onReady: undefined,
@description callback when main script processing and initial rendering has finished
onClose: undefined,
@description callback when list area closes
onOpen: undefined
@description callback when list area opens
