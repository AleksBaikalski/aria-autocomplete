Aria Autocomplete On GitHub Accessible, extensible, plain JavaScript autocomplete with multi-select

Examples

Array as Source, using default options

When the source is an Array, it can either be an Array of Strings:

new AriaAutocomplete(element, {
    source: [
        "Afghanistan",
        "Albania",
        "Algeria",
        ...more
    ]
});

Or you can use an Array of Objects. The objects used should have value and/or label properties. But if you're unable to transform them ahead of use, there is also a sourceMapping option.

By default, the autocomplete will filter by only the label property, but you can add an Array of Strings to the alsoSearchIn option to indicate other properties to filter by. The following example filters by the label and code3 properties, so searching for SGS will return the option for "South Georgia and the South Sandwich Islands" for example.

new AriaAutocomplete(element, {
    source: [
        {
            code: "AF",
            code3: "AFG",
            name:" Afghanistan"
        }, 
        {
            code: "AL",
            code3: "ALB",
            name: "Albania"
        },
        ...more
    ],
    sourceMapping: { 'value': 'code3', 'label': 'name' },
    alsoSearchIn: ['code3']
});

The autocomplete typically expects an Array source that it will transform for use internally. To be as extensible as possible however, Function, Endpoint, and Element sources are also supported:

Element(s) as Source, with Multi-select, autogrow, results limit, and show all control

If the source option is falsy, or is an empty Array, the autocomplete will check to see if the element used is a <select>, or if there are any checkboxes inside the element, and will build up a source Array from what's available.

This example uses a <select>, so the multiple option is automatically set to match its own state (true in this case). If checkboxes are used, the multiple option will always be set to true.

When an option in the autocomplete is selected, the corresponding <option> or checkbox will be set to selected / checked as well.

new AriaAutocomplete(element, {
    deleteOnBackspace: true,
    showAllControl: true,
    autoGrow: true,
    maxResults: 50,
    minLength: 0,
    maxItems: 5
});

Function as Source, with Multi-select, but without autogrow

If you want to use your own client-side filtering logic, you can also use a function as the source - it will be passed the search term, and a render function, which expects an Array to be passed to it (using either of the formats mentioned above).

If the autocomplete is initialised on an input, which has a starting value, the source function will also be called immediately with that value passed to it. In this case, the render function expects an Array of possible entries to check that starting value against, so that it can internally determine what the starting label (or multi-select elements) should be.

This example searches based on each country's 3 letter country code, instead of its name, e.g. "DZA" for "Algeria", or "GBR" for the United Kingdom. The onItemRender callback is also being used to display the code next to the country name in the results.

The initialised input's starting value is "GLP,ZWE" - the country codes for Guadeloupe and Zimbabwe respectively - so this will also initialise with two values already selected.

new AriaAutocomplete(element, {
    maxItems: 4,
    maxResults: 10,
    multiple: true,
    source: function (query, render) {
        let result = [];
        let isStartingCall = this === window;
        let value = query.toLowerCase().trim();
        let arrToCheck = isStartingCall ? value.split(',') : [value];
        countryListAllIsoData.forEach(entry => {
            let code3 = entry['code3'].toLowerCase();
            let addIt = false;
            arrToCheck.forEach(val => {
                if (code3.search(val) !== -1) {
                    addIt = true;
                }
            });
            if (addIt) {
                result.push(entry);
            }
        });
        render(result);
    },
    onItemRender: itemData => `${itemData.label} (${itemData.code3})`,
    sourceMapping: { 'value': 'code3', 'label': 'name' }
});

Despite the custom source Function, this example still needs a sourceMapping option, because the label is used when rendering the list, and the value is used when filling the original <input />

String (endpoint) as Source

If you want to send the query to an endpoint / your server, you can do that too. The current query, and your chosen value for max results to render, will be added to the url using the asyncQueryParam and asyncMaxResultsParam respectively.
E.g. https://some-url-here/endpoint?q=norway&limit=20.
You can also use the onAsyncPrep callback to modify the URL.

As with a Function as the source, if the autocomplete is initialised on an input with a starting value, the endpoint will be called immediately.

This example has no filtering in the request, as it's just requesting a JSON file, so it's doing some basic filtering in the onAsyncSuccess callback. The initialised input's starting value is "CAN" (the code3 property for Canada).

new AriaAutocomplete(element, {
    source: 'countrylistallisodata.json',
    sourceMapping: { 'value': 'code3', 'label': 'name' },
    onAsyncPrep: url => url,
    onAsyncSuccess: function (query, xhr) {
        let result = [];
        let isStartingCall = this === window;
        let value = query.toLowerCase().trim();
        let response = JSON.parse(xhr.responseText);
        let propToCheck = isStartingCall ? 'value' : 'name';
        response.forEach(entry => {
            if (entry[propToCheck].toLowerCase().search(value) !== -1) {
                result.push(entry);
            }
        });
        return result;
    },
    asyncMaxResultsParam: 'limit',
    asyncQueryParam: 'q',
    maxResults: 25
});

All component options that accept a Function will have their context (this) set to include the full autocomplete API (assuming you use an ordinary function expression instead of arrow functions). The only exception to this is the Function and Endpoint starting cases mentioned above, where the context is not manually set. The isStartingCall variable in the above examples is used to illustrate this.

Options

The full list of available options is as follows:

Core options

name: string
Give the autocomplete a name so that (in single-select mode) its value will be included in form submissions.
(Instead of using this option, I would advise initialising the autocomplete on an existing input that will be submitted; this approach is compatible with the control in multiple mode)
source: string[] | Object[] | string | Function
string for async endpoint, array of strings, array of objects with value and label, or function
sourceMapping: Object
an object detailing the properties to use for the label and value when using an Array of Objects as the source - see Array examples above.
alsoSearchIn: String[]
Additional properties to use when searching for a match
delay: number
input delay before running a search
Default: 100
minLength: number
Minimum number of characters to run a search (includes spaces)
Default: 1
maxResults: number
Maximum number of results to render
Default: 9999
showAllControl: Boolean
Render a button that triggers showing all options
Default: false
confirmOnBlur: Boolean
Confirm current selection (from using arrow keys) when blurring off of the control. Will also check for an exact text match
Default: true
multiple: Boolean
Allow multiple items to be selected
Default: false
autoGrow: Boolean
Set input width to match its content
Default: false
maxItems: number
Maximum number of items that can be selected in multiple mode
Default: 9999
multipleSeparator: string
In multiple mode, if the original element was an input, the character that separates the values
Default: `,`
deleteOnBackspace: Boolean
If input is empty and in multiple mode, delete last selected item on backspace
Default: false

Async mode options

asyncQueryParam: string
When source is a string, param to use when adding input value to it
Default: `q`
asyncMaxResultsParam: string
When source is a string, param to use when adding maxResults option (+ current selected count in multiple mode) to it
Default: `limit`

Styling / rendering options

placeholder: string
Placeholder text to show in generated input
Default: ``
noResultsText: string
Text to show (and announce to screen readers) if no results are found. If this is an empty string, the list will close instead
Default: `No results`
cssNameSpace: string
The string to prepend to all main classes for BEM naming e.g. `aria-autocomplete__input`
(Some other generic class-names are used alongside these as well, such as `disabled`, `focused`, `hidden`, etc.)
Default: `aria-autocomplete`
listClassName: string
Custom class name to add to the generated list element
Default: ``
inputClassName: string
Custom class name to add to the generated input element
Default: ``
wrapperClassName: string
Custom class name to add to the generated component wrapper
Default: ``

Screen reader enhancements

srDeleteText: string
In multi mode, screen reader text used for element deletion - prepended to label
Default: `delete`
srDeletedText: string
Screen reader text announced after deletion - appended to label
Default: `deleted`
srSelectedText: string
Screen reader text announced after selection - appended to label
Default: `selected`
srListLabelText: string
Screen reader explainer added to the list element via aria-label attribute
Default: `Search suggestions`
srAssistiveText: string
Screen reader description used for main input when empty
Default: `When results are available use up and down arrows to review and enter to select. Touch device users, explore by touch or with swipe gestures.`
srResultsText: Function
Screen reader announcement after results are rendered
Default: length => `${length} ${length === 1 ? 'result' : 'results'} available.`

Callbacks

onSearch: Function
Before search is performed - can be used to affect search value by returning a new one
Default: query => query
onAsyncPrep: Function
Callback before async call is made - receives the URL. Can be used to format the endpoint URL by returning a String
Default: url => url
onAsyncSuccess: Function
After async call completes - is passed the value and the XHR Object. Can be used to format the results by returning an Array
Default: (query, xhr) => xhr.responseText
onItemRender: Function
Called for each item rendered in the list - Can be used to format the <li> content by returning a String
Default: itemData => itemData.label
onResponse: Function
Prior to rendering - can be used to format the results by returning an Array
Default: optionsToRender => optionsToRender
onConfirm: Function
After an autocomplete selection is made
Default: selection => {}
onDelete: Function
After an autocomplete selection is deleted (programmatically in single-select mode, or by user action in multi-select mode)
Default: selection => {}
onReady: Function
When main script processing and initial rendering has finished
Default: componentWrapper => {}
onClose: Function
When list area closes
Default: listElement => {}
onOpen: Function
When list area opens
Default: listElement => {}
